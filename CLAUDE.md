## Build Commands

**IMPORTANT: Always use `just` tasks to build or test. Never invoke `cmake`, `make`, or compiler commands directly.**

This project uses CMake. All source lives in `neo/`, and builds output to `build/` at the repo root.

**Using the justfile (recommended for macOS development):**
```bash
just gen      # cmake -S neo --preset debug
just build    # gen + cmake --build build
just run      # build + launch dhewm3.app with game assets
just clean    # remove build/
just assets   # download game demo assets
```

**Manual CMake:**
```bash
mkdir build && cd build
cmake ../neo/
make -j$(nproc)
```

**macOS with Homebrew OpenAL Soft (required — system OpenAL is unsupported):**
```bash
cmake -DOPENAL_LIBRARY="/opt/homebrew/opt/openal-soft/lib/libopenal.dylib" \
      -DOPENAL_INCLUDE_DIR="/opt/homebrew/opt/openal-soft/include" \
      ../neo/
```

**Using CMake presets (defined in `neo/CMakePresets.json`):**
```bash
cmake --preset debug    # debug build to ../build
cmake --preset release  # release build to ../build/release
cmake --build build
```

**Key CMake options:**
| Option | Default | Description |
|--------|---------|-------------|
| `CORE` | ON | Build the engine |
| `BASE` | ON | Build base game (base.so/dylib/dll) |
| `D3XP` | ON | Build RoE expansion (d3xp.so/dylib/dll) |
| `DEDICATED` | OFF | Build dedicated server |
| `TOOLS` | OFF | Build editor tools (Windows only) |
| `SDL2`/`SDL3` | ON/OFF | SDL version selection |
| `IMGUI` | ON | Settings menu via Dear ImGui |
| `HARDLINK_GAME` | OFF | Compile game code into executable |
| `ASAN`/`UBSAN` | OFF | Sanitizers (GCC/Clang only) |
| `TRACY` | OFF | Tracy profiler integration |

**Windows (MSVC):**
```bash
cmake -G "Visual Studio 17 2022" -A x64 \
      -DDHEWM3LIBS=/path/to/dhewm3-libs/x86_64-w64-mingw32 \
      -DDEDICATED=ON -DTOOLS=ON \
      -S neo -B build
cmake --build build --config RelWithDebInfo
```

There is no unit test suite. CI runs full builds via GitHub Actions (`.github/workflows/`) for macOS, Linux, and Windows. Verify changes by building successfully and running the game.

## Architecture

dhewm3 is a source port of Doom 3, organized as an engine executable that loads game logic as shared libraries (DLLs) at runtime.

### Build Outputs
- **`dhewm3`** — Main game executable (renderer, audio, UI, physics, networking)
- **`dhewm3ded`** — Dedicated server (no GL/OpenAL; uses stubs in `sys/stub/`)
- **`base.so/.dylib/.dll`** — Base game logic (entity system, AI, scripting VM, physics)
- **`d3xp.so/.dylib/.dll`** — Resurrection of Evil expansion logic

### Source Layout (`neo/`)

| Directory | Purpose |
|-----------|---------|
| `idlib/` | Core library: containers, math (SIMD), geometry, hashing, strings |
| `framework/` | Engine framework: filesystem, console, CVars, commands, networking async, ZIP |
| `renderer/` | OpenGL renderer: materials, images, models, lighting, stencil shadows |
| `sound/` | OpenAL audio: caching, OGG decoding (stb_vorbis), EFX reverb |
| `ui/` | GUI scripting system: menus, HUD, bind windows |
| `cm/` | Collision model: traces, contacts, debug draw |
| `game/` | Base game code: entities, AI (AAS), animation, physics, script compiler/VM |
| `d3xp/` | RoE expansion code (nearly identical structure to `game/`) |
| `sys/` | Platform abstraction: `posix/`, `linux/`, `osx/`, `win32/`, `aros/`, `stub/` |
| `tools/` | Editor tools: dmap compiler, AAS compiler, Radiant, material editor (Windows only) |
| `libs/imgui/` | Dear ImGui v1.92.5 (settings menu, F10 in-game) |

### Key Subsystems

**Entity/Game System** (`game/` and `d3xp/`): Class-based entity hierarchy with a custom scripting language. The `gamesys/` subdirectory contains the event dispatch system, script VM, class reflection, and save game support.

**Renderer**: OpenGL with ARB extensions. Uses stencil shadows ("Carmack's Reverse"). Material system driven by `.mtr` declaration files. Supports S3TC/DXT and BC7 texture compression.

**Platform Layer** (`sys/`): OS-specific code is fully isolated here. `sys/platform.h` defines architecture/OS detection macros (`D3_ARCH`, `D3_OSTYPE`, `D3_SIZEOFPTR`, `D3_IS_BIG_ENDIAN`). The dedicated server stubs out GL and OpenAL via `sys/stub/`.

**ImGui Settings Menu**: Optional (enabled by `IMGUI=ON`, requires SDL2/SDL3 and C++11). Provides in-game settings UI accessible via F10, including video, audio, and control bindings.

### Code Conventions

- Class names use `id` prefix with CamelCase: `idStr`, `idVec3`, `idEntity`
- No C++ namespaces (original Doom 3 style)
- Platform-agnostic code in main subsystems; all OS-specific code in `sys/`
- Use `idStr` instead of raw `strcmp`/`strncmp` (see warning in `idlib/Str.h`)
- All source files carry GPL license headers
- `-ffp-contract=off` is required for correctness of math code (FMA disabled)
- C++98/03 compatible by default; C++11 only when `IMGUI=ON`
